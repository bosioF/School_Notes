## Architettura Client / Server

### Client

- È il **browser**
    
- Mostra l’interfaccia all’utente
    
- Invia richieste al server (HTTP)
    
- Usa **JavaScript vanilla**, HTML, CSS
    

### Server

- È il **backend**
    
- Riceve le richieste
    
- Elabora i dati (logica applicativa)
    
- Comunica con database / API
    
- Risponde al client (JSON, HTML, ecc.)
    

---

## Client-side (JavaScript Vanilla)

### Ruolo

- Gestione eventi (click, submit, input)
    
- Manipolazione DOM
    
- Invio richieste HTTP
    
- Ricezione e visualizzazione dati
    

---

## Server-side (Node.js)

### Cos’è Node.js

- Runtime JavaScript lato server
    
- Basato su **event loop** (asincrono, non bloccante)
    
- Usa V8 (motore JS di Chrome)
    

### Compiti del server

- Gestione rotte (endpoint)
    
- Logica di business
    
- Autenticazione
    
- Accesso database
    
- Invio risposte HTTP
    

---

## Vantaggi Client/Server

Separazione delle responsabilità  
Scalabilità  
Manutenzione più semplice  
Riutilizzo API (web, mobile)
